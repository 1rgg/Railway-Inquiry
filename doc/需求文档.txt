火车中转方案查询静态网页 开发需求文档
一、项目概述
项目目标：开发一个纯前端静态网页，实现火车换乘中转方案查询功能。用户可输入出发地、目的地、出发日期，查询对应的火车换乘方案、中转信息及席位余票情况。
实现形式：纯静态 HTML 页面，无需后端服务，可直接在浏览器打开或部署到任意静态服务器。
设计风格：简约清晰，信息层级明确，去除冗余视觉元素，聚焦查询功能与结果展示，保证信息易读、操作简便。
二、核心功能需求
2.1 查询参数输入模块
页面顶部设置查询表单，用户填写以下条件后触发查询：
出发站点：文本输入框，必填，对应接口参数 add，支持输入城市或车站名称
目的站点：文本输入框，必填，对应接口参数 end，支持输入城市或车站名称
出发日期：日期选择器，必填，选择后自动拆解为年 (y)、月 (m)、日 (d) 三个接口参数；日期可选范围限制为当日起 15 天内，匹配接口预售规则
查询按钮：点击后执行参数校验与接口请求
配置说明：接口鉴权参数 id（开发者 ID）与 key（开发者 KEY）集中在 JS 代码头部配置，无需用户在前台输入；代码中预留注释，提示开发者替换为自身的 ID 与密钥。
2.2 查询交互与状态反馈
参数校验：点击查询时校验必填项是否为空，为空时在对应输入框下方给出文字提示，阻止请求发起。
加载状态：接口请求过程中，在结果区域展示加载提示，禁用查询按钮，避免重复提交。
异常处理：
接口返回 code=400 时，直接展示返回的 msg 错误信息（如 “通讯秘钥错误”）
网络请求失败时，展示 “网络异常，请检查网络后重试” 提示
查询无结果时，展示 “暂无符合条件的换乘方案” 占位提示
2.3 换乘方案结果展示
接口请求成功（code=200）后，以卡片列表形式展示换乘方案，按接口返回顺序排列，单张卡片包含以下信息：
方案核心信息（卡片头部）
全程总历时（取 datas.lishi），突出展示
出发站 + 发车时间（datas.addname + datas.addtime）
到达站 + 到达时间（datas.endname + datas.endtime）
标识是否同站换乘：hcst=0 显示 “同站换乘”，hcst=1 显示 “异站换乘”
中转信息（卡片中部）
中转站名称（datas.zzname）
中转等待时长（datas.zzwait）
车次与余票概览
展示两段行程的车次编号（datas.list[0].code、datas.list[1].code）
默认展示二等座 / 硬座余票状态，优先展示高频查询席位
详情展开
点击卡片可展开完整行程详情，展示每一段车次的出发 / 到达站点、时间、历时，以及全席位余票信息
余票展示规则：数字显示具体票数、“有” 显示有票、“无” 显示无票、-- 显示为 “无该席位”
标注是否支持身份证检票（scard=1 显示 “支持身份证检票”）
2.4 分页功能
页面底部设置分页控件，默认展示第 1 页
支持 “上一页”“下一页” 切换，切换页码时自动重新请求接口，同步更新结果
页码对应接口参数 page
三、接口对接规范
3.1 基础请求规则
请求编码：UTF-8
请求方式：POST（禁止使用 GET，避免参数过长与中文乱码）
提交格式：application/x-www-form-urlencoded（表单格式）
接口地址：在代码头部预留配置项，由开发者填写实际接口地址
3.2 请求参数清单
表格
参数名	类型	必填	说明	取值来源
id	string	是	开发者数字 ID	代码全局配置
key	string	是	开发者通讯密钥	代码全局配置
add	string	是	出发站点名称	用户输入
end	string	是	目的站点名称	用户输入
y	string	是	出发年份	日期选择器拆分
m	string	是	出发月份	日期选择器拆分
d	string	是	出发日期	日期选择器拆分
page	number	否	查询页码，默认 1	分页控件
3.3 返回数据核心字段映射
基于接口返回 JSON 结构，核心使用字段如下：
顶层字段：code（状态码）、msg（提示信息）、page（当前页码）、datas（换乘方案数组）
单条换乘方案（datas 子项）核心字段：
行程概览：lishi（全程历时）、addname/addtime、endname/endtime、hcst（是否同站）
中转信息：zzname（中转站）、zzwait（等待时长）
行程详情：list 数组，包含每段车次的编号、站点、时间、历时、各席位余票、scard（身份证检票）
四、界面设计要求
整体布局
页面居中约束最大宽度（建议 1200px），适配桌面端与移动端
顶部为页面标题 “火车中转方案查询”
中部为查询表单区，桌面端横向排布，移动端自动纵向堆叠
下部为结果展示区，未查询时显示简洁占位提示
视觉风格
主色调采用低饱和度蓝色，搭配中性灰色系，整体清爽克制
结果卡片使用白色背景、轻微阴影、小圆角，清晰区分不同换乘方案
文字层级分明：标题加粗，时间、历时等核心信息字号稍大，辅助说明文字使用浅灰色
无多余装饰元素，所有视觉设计服务于信息可读性
响应式适配
桌面端：表单横向排列，结果卡片单列展示，信息舒展
移动端（宽度 < 768px）：表单纵向堆叠，结果卡片自适应屏幕宽度，保证文字清晰可读
五、技术实现要求
技术栈：原生 HTML + CSS + JavaScript，不依赖重型前端框架，保证轻量、加载快；使用原生 Fetch API 发起网络请求。
代码规范：结构清晰，关键逻辑添加注释；接口地址、ID、密钥等可配置项集中管理，方便后续修改维护。
兼容性：兼容 Chrome、Edge、Firefox、Safari 等主流现代浏览器。
交付形式：最终产出为可直接运行的静态文件（HTML/CSS/JS），双击即可打开使用，也可部署到任意静态 Web 服务器。
六、补充说明
日期选择器默认选中当日，禁止选择今日之前的日期，以及超出 15 天预售期的日期。
余票字段展示严格匹配接口返回值：数字、“有”、“无”、“--” 分别对应不同状态，不做额外臆造。
字段适配以接口返回示例为准，确保所有展示信息与接口字段一一对应。
